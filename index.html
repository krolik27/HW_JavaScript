  4. * Самостоятельно разобраться с атрибутами тега script (async и defer)
    Решение

    Если браузер видит тег script, то он по стандарту обязан сначала выполнить его, а потом показать оставшуюся часть страницы.

    Можно поставить все подобные скрипты в конец страницы – это уменьшит проблему, но не избавит от неё полностью, если скриптов несколько. Решить эту проблему помогут атрибуты async или defer.

    Aтрибут async

    • Поддерживается всеми браузерами, кроме IE9
    • Скрипт выполняется полностью асинхронно. Браузер не останавливает обработку страницы, а спокойно работает дальше. Когда скрипт будет загружен – он выполнится.
    • Первым сработает тот скрипт, который раньше загрузится.
    • Скрипт async.js выполнится, как только загрузится – возможно, до того, как весь документ готов.

    Атрибут defer

    • Поддерживается всеми браузерами, включая самые старые IE
    • Скрипт выполняется полностью асинхронно. Браузер не останавливает обработку страницы, а спокойно работает дальше. Когда скрипт будет загружен – он выполнится.
    • Браузер гарантирует, что относительный порядок скриптов с defer будет сохранён.
    • Скрипт с defer сработает, когда весь HTML-документ будет обработан браузером, т.е. ждёт готовности всего документа.

Урок 2. Основные операторы JavaScript

  1. Дан код:
    var a = 1, b = 1, c, d;
    c = ++a; alert(c); // 2
    d = b++; alert(d); // 1
    c = (2+ ++a); alert(c); // 5
    d = (2+ b++); alert(d); // 4
    alert(a); // 3
    alert(b); // 3
    Почему код даёт именно такие результаты?
    Решение

    Инкремент ++ увеличивает на 1.
    Постфиксная форма i++ отличается от префиксной ++i тем, что возвращает старое значение, бывшее до увеличения.
    c = ++a; alert(c); // 2 - префиксная форма
    d = b++; alert(d); // 1 - постфиксная форма
    c = (2+ ++a); alert(c); // 5, где c=2+1+a(уже увеличенная на а+1, т.е. 2)
    d = (2+ b++); alert(d); // 4, где d=2+1+b(уже увеличенная на b+1, но в постфиксной форме, которая возвращает значение до увеличения, т.е.1)
    alert(a); // 3, где a=(1+(1+a))
    alert(b); // 3, где b=(1+1(+b)) - если результат оператора не используется, то постфиксная форма так же увеличивает значение переменной, как префиксная.

        var pow =parseInt(prompt("Введите степень"));

    Но надо решение с помощью рекурсии, тогда будет так 
     
        function power(val, pow){
          if (pow>0&&pow!==1){ //если степень не равна 1 и больше нуля
            return (val*power(val,pow-1));
          } else if (pow==0){ //если степень равна нулю
             return 1; 
          } else if (pow<0){ //если степень меньше нуля
            return (1/(val*power(val,Math.abs(pow)-1))); 
          } else { //если степень равна 1 
             return val;
          }
        }
        alert (power(val, pow));
